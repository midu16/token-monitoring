# Contributors

Thank you for contributing to the Token Monitoring Service! 🚀

We appreciate all the efforts made by developers around the world to improve this project.

## How to Contribute

If you would like to contribute to this project, please follow these steps:

1.  **Fork the repository** to your own GitHub account.
2.  **Create a feature branch** from `main` (`git checkout -b feature/your-feature-name`).
3.  **Implement your changes**. Ensure you follow the existing coding patterns and add tests where appropriate.
4.  **Run tests** to ensure everything is working as expected (`go test ./...`).
5.  **Commit your changes** with clear, descriptive commit messages.
6.  **Push to your fork**.
7.  **Open a Pull Request** against the `main` branch of the original repository.

## Code Standards

*   Follow standard Go idioms and formatting (`go fmt`).
*   Ensure all new features are accompanied by unit tests.
*   Keep commit messages concise and meaningful.

## Contact

If you have any questions or issues, please open an issue in the repository.
